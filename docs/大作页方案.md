# 电商商品图片识别打标完整落地方案

## 一、整体架构定位

### 硬件约束

单台 RTX 4060（8G 显存）部署全部本地模型与图像处理工具，不使用多卡、不引入重型大模型服务；

### 分层分工

1. 本地主力模型：Qwen2-VL-7B-Instruct INT4 AWQ 量化

   承载 90% 标准白底商品图，无任何工具调用逻辑，直接输出标准化商品标签；

2. 决策大脑：DeepSeek-VL API（仅疑难图分流）

   唯一负责工具调用决策，自主判断调用哪些本地图像处理工具；

3. 本地轻量化工具集（极简、4060 可同时承载，无重型模型）

   仅保留电商刚需 4 类工具，全部封装 FastAPI 内网接口，供 DeepSeek 远程调用；

4. 分流规则：本地模型置信度不足→送入 DeepSeek，由 DeepSeek 调度本地工具预处理图片后再解析。

## 二、各模块详细说明

### 1. 本地主力识别模型（Qwen2-VL-7B INT4 AWQ）

1. 训练：LLaMA Factory QLoRA 微调，自有电商商品图文数据集，仅训练标签抽取能力，不做工具指令微调；
2. 部署：vLLM INT4 AWQ 量化推理，单 4060 可稳定批量推理；
3. 处理范围：白底、单商品、无水印、画面清晰；
4. 输出：固定格式类目、颜色、材质、风格标签；
5. 核心规则：这条链路**不接入任何工具、不发起外部接口请求**，追求最低延迟、最高吞吐。

### 2. 决策大脑 DeepSeek-VL API（工具调用唯一载体）

1. 触发条件：
   - 图片多物体、场景实拍、背景杂乱；
   - 图片模糊、反光、存在大面积水印 / 遮挡文字；
   - 本地模型输出置信度＜0.75；
2. 核心能力：原生 Function Calling，自动判断工具组合、支持多轮调用；
3. 调用逻辑：DeepSeek 输出工具调用指令 → 业务中台拦截请求 → 内网调用本地工具服务 → 处理后图片回传给 DeepSeek 二次识别；
4. 收尾统一：DeepSeek 生成商品自然描述后，回传给本地 Qwen2-VL 做标签标准化收口，保证全平台标签格式统一。

### 3. 本地轻量化工具集（2 套极简工具，4060 统一部署）

全部开源轻量模型，Docker 独立封装，共用单卡显存调度，无复杂依赖，仅保留电商刚需功能：

#### 工具 1：商品抠图 RMBG-2.0（轻量版）

- 功能：去除复杂场景背景，生成纯白底商品图；
- 触发场景：DeepSeek 识别背景杂乱、环境实拍图；
- 轻量化优化：输入尺寸限制 768，CPU/GPU 混合推理，显存占用＜1G。

#### 工具 3：商品目标检测 YOLOv8-Nano（基于自有商品数据微调）

- 功能：框选出图内每件独立商品，返回坐标并自动裁切子图；
- 触发场景：一张图片多件穿搭、套装、多品类商品同框；
- 轻量化：nano 极小模型，显存占用极低，专门适配电商服饰 / 3C 类目。


### 4. 业务中转中台（核心调度层）

1. 拦截 DeepSeek API 下发的所有工具调用指令；
2. 转发至内网本地工具接口（RMBG/LaMa/YOLO/OCR）；
3. 统一管理图片临时存储，处理后图片生成内网可访问地址回传给 DeepSeek；
4. 降级策略：任意本地工具超时 / 显存不足，自动跳过预处理，直接原图识别，不阻断主流程；
5. 数据回流：工具清洗后的高清白底图定时存入训练集，LLaMA Factory 增量微调本地 Qwen2-VL，长期降低疑难图流量。

## 三、完整业务执行链路

```
商家/用户上传商品图片
        ↓
图片统一压缩预处理（Pillow/OpenCV，CPU执行）
        ↓
本地Qwen2-VL INT4 AWQ推理（无工具调用）
        ↓
判断分支
分支A：置信度≥0.75、单商品白底清晰图
        → 直接输出标准化商品标签 → 入库，流程结束
分支B：置信度＜0.75 / 多物体/杂乱背景/水印模糊
        → 送入DeepSeek-VL API（工具调用决策大脑）
                ↓
        DeepSeek自主判断需要调用哪些本地工具，生成function call
                ↓
        业务中台拦截调用，内网请求RTX4060上部署的本地图像处理工具
                ↓
        工具完成抠图/单品裁切，返回处理后图片
                ↓
        清洗后的图片重新传入DeepSeek，完成商品完整语义解析
                ↓
        DeepSeek输出商品完整图文描述，回传给本地Qwen2-VL模型
                ↓
        本地模型统一转换成平台标准标签 → 标签+原图+清洗图入库
                ↓
        优质清洗图片定时同步至训练数据集，增量微调本地模型
```

## 四、资源负载说明（RTX4060 8G 适配保障）

1. 常驻显存占用拆分
   - Qwen2-VL-7B INT4 AWQ：约 4-4.5GB；
   - 4 套轻量工具分时复用剩余 3.5-4GB 显存，不同时做高并发推理；
2. 并发限制：单 4060 限制本地识别并发≤4，工具处理并发≤2；
3. 优化手段：所有工具限制输入图片最大边长 768，禁用高清大图推理，降低显存峰值。

## 五、关键设计约束（贴合你的需求）

1. 本地模型完全不做工具调用

   简单图片链路无任何外部接口、无多轮循环，保证速度与并发；工具调用逻辑全部隔离在 DeepSeek 分支；

2. 工具全部本地私有化部署

   预处理图片不传给第三方商用抠图 / 检测 API，新品、买家秀素材不外流；

3. 工具极简克制，仅 4 个刚需工具，不增加复杂视觉模型，适配单 4060；

4. 成本可控：90% 流量本地零 Token 费用，仅少量疑难图消耗 DeepSeek API；

5. 架构轻量化，无分布式集群、多机调度复杂逻辑，单机即可完整跑通。

## 六、可选扩展（不改动现有架构，按需开启）

1. 若后期完全私有化、禁用所有云端 API：

   将 Qwen2-VL 改造增加工具调用 SFT 微调，替代 DeepSeek 作为决策大脑；

2. 若流量上涨：新增一台 4060 做工具服务负载分离；

3. 风控拓展：OCR 提取文字后对接本地违禁词库，无需新增图像工具。
